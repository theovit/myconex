## 1. Bot Token

The bot token is the secret credential that lets your code connect to Discord.

1. Open the Developer Portal → **Bot** (left sidebar)
2. Click **Reset Token** → confirm → copy the token immediately (it's only shown once)
3. Paste it into `.env`:

```
DISCORD_BOT_TOKEN=your_copied_token_here
```

> **Security:** Never commit the token to git. The `.gitignore` already excludes `.env`.
> If a token is ever exposed publicly, reset it immediately in the Portal.

## 2. Privileged Gateway Intents

Still on the **Bot** page, scroll to **Privileged Gateway Intents** and enable:

| Intent | Required for |
|--------|-------------|
| **Message Content Intent** | Reading message text (without this the bot sees empty strings) |
| **Server Members Intent** | Resolving usernames in `DISCORD_ALLOWED_USERS` to numeric IDs |
| Presence Intent | *Optional — not used by MYCONEX* |

Click **Save Changes** after toggling each intent.

> These must be enabled in the Portal **and** in code. The gateway already sets
> `intents.message_content = True` and `intents.members = True` in
> `core/gateway/discord_gateway.py`.

## 3. OAuth2 Scopes & Bot Permissions

Navigate to **OAuth2 → URL Generator** (left sidebar).

### 3.1 Scopes

Select both:

- `bot` — allows the application to join servers as a bot user
- `applications.commands` — allows the bot to register and use slash commands

### 3.2 Bot Permissions

With `bot` scope selected, the **Bot Permissions** panel appears below. Check:

| Permission | Why needed |
|------------|-----------|
| **Send Messages** | Post responses in text channels |
| **Send Messages in Threads** | Post responses inside Discord threads |
| **Read Message History** | Read context in channels (required for reliable message handling) |
| **View Channel** | See the channels the bot is added to |
| **Use Application Commands** | Run slash commands (`/ask`, `/status`, etc.) |
| **Embed Links** | Render URLs as embeds in responses |
| **Attach Files** | Send file attachments if needed |
| **Add Reactions** | Optional — for reaction-based feedback |
| **Manage Messages** | Optional — allows bot to delete/pin messages if needed later |

#### Resulting permission integer

The permissions above produce the integer: **`414464683072`**

You can paste this directly into the invite URL (see Section 4).

### 3.3 Scopes integer

For manual URL construction, the scope string is: `bot%20applications.commands`

```
https://discord.com/oauth2/authorize
  ?client_id=1469408384070586432
  &permissions=414464683072
  &scope=bot%20applications.commands
```

Single-line form (copy-paste ready):

```
https://discord.com/oauth2/authorize?client_id=1469408384070586432&permissions=414464683072&scope=bot%20applications.commands
```

Open this URL in a browser while logged in to Discord, choose your server, and
click **Authorise**. The bot will appear in your server's member list (offline
until you start it).

## 5. Slash Command Sync

Slash commands (`/ask`, `/reset`, `/status`, `/tier`) are registered automatically
when the bot starts: the `on_ready` event calls `bot.tree.sync()`.

### Global vs. Guild sync

| Type | Propagation time | Use case |
|------|-----------------|----------|
| **Global** (default) | Up to 1 hour | Production — commands appear in every server |
| **Guild** (instant) | ~seconds | Development / testing |

To force an instant guild sync during development, run this snippet once while
the bot is connected:

```python
# run from a Python shell with the bot running
import asyncio, discord

# Replace with your bot token and guild ID
TOKEN = "your_bot_token"
GUILD_ID = 123456789012345678   # right-click your server → Copy Server ID

intents = discord.Intents.default()
bot = discord.ext.commands.Bot(command_prefix="!", intents=intents)

@bot.event
async def on_ready():
    guild = discord.Object(id=GUILD_ID)
    bot.tree.copy_global_to(guild=guild)
    synced = await bot.tree.sync(guild=guild)
    print(f"Synced {len(synced)} command(s) to guild {GUILD_ID}")
    await bot.close()

asyncio.run(bot.start(TOKEN))
```

Or — more simply — just wait up to 1 hour after first run for global sync to propagate.

## 7. Enabling the Bot in MYCONEX

### Option A — config flag (recommended for production)

In `config/mesh_config.yaml`:

```yaml
discord:
  enabled: true
  application_id: "1469408384070586432"
  require_mention: false    # false = responds to all messages in configured channels
  auto_thread: false        # true = each @mention spawns its own thread
```

Start normally:

```bash
python main.py
```

### Option B — CLI mode flag

```bash
# Discord only (no REST API)
python main.py --mode discord

# Discord + REST API + mesh
python main.py --mode full
```

### Option C — environment variable override

```bash
DISCORD_BOT_TOKEN=your_token python main.py --mode discord
```

## 8. Runtime Behaviour Reference

| Env var | Default | Effect |
|---------|---------|--------|
| `DISCORD_BOT_TOKEN` | *(required)* | Bot authentication token |
| `DISCORD_APPLICATION_ID` | `1469408384070586432` | Application ID for slash command registration |
| `DISCORD_REQUIRE_MENTION` | `false` | `true` = bot only responds when @mentioned in servers |
| `DISCORD_FREE_RESPONSE_CHANNELS` | *(empty)* | Comma-separated channel IDs that bypass `require_mention` |
| `DISCORD_AUTO_THREAD` | `false` | `true` = auto-create thread on @mention in text channels |
| `DISCORD_ALLOWED_USERS` | *(empty = all)* | Comma-separated Discord user IDs; empty allows everyone |
| `DISCORD_ALLOW_BOTS` | `none` | `none` = ignore other bots, `all` = respond to all |

## 9. Verifying the Setup

Once the bot token is set and the bot is started:

1. **Bot appears Online** in your Discord server's member list
2. **Slash commands visible** — type `/` in any channel; MYCONEX commands appear
3. **Test with `/tier`** — should return `Node tier: T2` (or whatever tier this node is)
4. **Test with `/ask What is MYCONEX?`** — should route through the TaskRouter and return an AI response
5. **Test in DM** — DM the bot directly; it should respond without needing a mention
6. **Check logs** — `tail -f /var/log/myconex/myconex.log` for connection and routing events

## 10. Troubleshooting

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| Bot stays offline | Token wrong or not loaded | Check `.env` has correct `DISCORD_BOT_TOKEN`; check `load_dotenv` runs |
| Slash commands missing | Not synced yet | Wait up to 1 hr, or use guild sync (Section 5) |
| Bot sees empty message content | Message Content Intent not enabled | Enable in Portal → Bot → Privileged Gateway Intents |
| Bot ignores messages | `require_mention=true` and user didn't @mention | Set `require_mention: false` in `mesh_config.yaml` or @mention the bot |
| Username resolution fails | Server Members Intent not enabled | Enable in Portal → Bot → Privileged Gateway Intents |
| `DISCORD_BOT_TOKEN not set` in logs | `.env` not loaded or token placeholder not replaced | Edit `.env`, replace `REPLACE_WITH_YOUR_BOT_TOKEN` |
| `403 Missing Permissions` | Bot missing a required permission | Re-invite with correct permissions using URL from Section 4 |
| Slash commands say "This interaction failed" | Bot crashed during handler | Check logs; usually a TaskRouter or Ollama connectivity issue |
